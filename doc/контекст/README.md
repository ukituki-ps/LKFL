# Контекст

> **🗺️ Навигация:** [`doc/NAVIGATION.md`](../NAVIGATION.md) — карта «вопрос → файл:строка»

Этот раздел описывает **философию**, **бизнес-проблему**, **акторов**, **ограничения** и **negative criteria** проекта «Платформа гибких льгот».

> **⚠ Читать первым [`философия.md`](./философия.md).** Она задаёт контекст для всех остальных файлов раздела.

## Назначение

Контекст — верхний уровень документации. Он отвечает на вопросы:
- **Что это за платформа?** — универсальная white-label multi-tenant система. СДЭК — первый tenant, не единственный
- **Какую проблему решаем?** — сотрудникам нужен удобный способ распределять бюджет льгот между продуктами
- **Кто участвует?** — сотрудники, администраторы, HR, провайдеры сервисов
- **В каких ограничениях работаем?** — 100 000+ пользователей, ФСТЭК, 152-ФЗ, сроки лета 2026
- **Что НЕ делаем?** — negative criteria, исключающие неподходящие подходы
- **Что настраивается без кода?** — матрица настраиваемости

## Поток информации

```
Контекст
  │  Философия → Всё            (платформа ≠ проект СДЭК)
  │  Проблема   → Архитектура   (что строить)
  │  Акторы     → Спецификация  (какие journeys)
  │  Огранич.   → Архитектура   (какой стек)
  │  Не делаем  → Спецификация  (negative criteria)
  │  Настройка  → Архитектура   (что конфигурируется, что — кодом)
```
