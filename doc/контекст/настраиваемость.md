## Модели биллинга — примеры

> **ADR-021:** `condition_cel` — CEL expression. `condition_source` — текст на русском для LLM.

Платформа поддерживает разные финансовые модели через комбинацию Billing Rules.

### Модель A: Баллы, ежемесячные, по грейдам (СДЭК)

```yaml
# HR вводит: "начисление 1000 баллов из grade-таблицы, каждый месяц"
Rule 1: direction=credit, amount_expr="grade_table[grade]", frequency=monthly, trigger=cron, condition_cel="user.status == 'active'"

# HR вводит: "списание 500 на фитнес, каждый месяц, активная подписка worldclass"
Rule 2: direction=debit, amount=500, category=fitness, frequency=monthly, trigger=cron, condition_cel="context.engagement_active && context.engagement_provider == 'worldclass'"

# HR вводит: "списание 3500 за ДМС при активации"
Rule 3: direction=debit, amount=3500, category=dms, frequency=one-time, trigger=event:engagement_debit
```

### Модель B: Рубли, ежеквартальные, фиксированные

```yaml
Rule 1: direction=credit, amount=5000, frequency=quarterly, trigger=cron, condition_cel="user.status == 'active'"
Rule 2: direction=debit, amount=1200, category=training, frequency=one-time, trigger=event:engagement_debit
Currency: RUB
```

### Модель C: Смешанная — баллы + софинансирование (engagement)

```yaml
Rule 1: direction=credit, amount=1000, frequency=monthly, trigger=cron, condition_cel="user.status == 'active'"

# HR вводит: "бонус 500 если ENPS >= 8"
Rule 2: direction=credit, amount=500, condition_cel="events.enps_score >= 8", trigger=event:engagement_credit

Rule 3: direction=debit, amount_expr="event.engagement_offer_cost * 0.7", category=any, trigger=event:engagement_debit
Rule 4: direction=debit, amount_expr="event.engagement_offer_cost * 0.3", category=cofinance, trigger=event:engagement_debit
```

### Модель D: Баллы с бонусом за стаж и рефералов

```yaml
Rule 1: direction=credit, amount=1000, frequency=monthly, trigger=cron, condition_cel="user.status == 'active'"

# HR вводит: "бонус 850 при стаже 3, 5, 10, 15 или 20 лет"
Rule 2: direction=credit, amount=850, condition_cel="user.years_of_service in [3, 5, 10, 15, 20]", trigger=event:anniversary_reached

# HR вводит: "бонус 300 если реферал зарегистрировался"
Rule 3: direction=credit, amount=300, condition_cel="context.referrer.status == 'registered'", trigger=event:referral

# HR вводит: "бонус 200 ежемесячно если есть дети"
Rule 4: direction=credit, amount=200, condition_cel="user.has_children", frequency=monthly, trigger=cron
```

## Модели энгейджментов — примеры

### Типы activity-энгейджментов по умолчанию (type=activity)

> **ADR-021:** step condition — CEL expression. `condition_source` → LLM Proxy → `condition_expr`.

| Slug | flow step | condition_expr (CEL) | condition_source (HR input) |
|-----|-----------|--------|---|
| `survey` | form → condition_check | `context.answers.size() >= N` | "заполнено не менее N ответов" |
| `event` | condition_check | `context.check_in == true` | "сотрудник пришёл на мероприятие" |
| `referral` | condition_check | `context.referred_user.status == "registered"` | "приглашённый зарегистрировался" |
| `anniversary` | condition_check | `user.years_of_service >= threshold` | "стаж достиг threshold лет" |
| `enps` | form → condition_check | `context.score_submitted == true` | "ENPS score отправлен" |
| `health_check` | form → condition_check | `context.form_completed == true` | "форма здоровья заполнена" |
| `custom` | condition_check | (CEL из LLM) | любой текст на русском |

### Как добавить новый тип activity-энгейджмента

1. HR открывает ЛК-2 → «Энгейджменты» → `POST /admin/engagement-types?type=activity`
2. Заполняет: slug, название, описание, `type="activity"`
3. Создаёт engagement flow с нужными steps (form, condition_check etc.)
4. Создаёт offer: billing_direction=credit, billing_rule_id, cost (бонус)
5. Сохраняет → type доступен в каталоге `?type=activity`

> **Примечание:** тип `survey` и `event` имеют встроенный UI-рендерер (SurveyForm, EventCheckin). Типы `custom` — требуют настройки UI-компонента на фронтенде (без изменения бэкенд-кода, через JSON-конфиг EngagementFlow.steps).
