# M05 — Унификация: единая абстракция «Энгейджмент»

## Описание

После M03 (пересборка льгот на 5 сущностей) выявлена топологическая эквивалентность модулей «Льготы» и «Активности». Обе системы имеют одинаковую структуру: тип → экземпляр → пользовательское выполнение → биллинг.

## Цель

Единая абстракция **Engagement** заменяет две параллельные модели:
- Льготы (Benefit, BenefitPlan, ActivationFlow, UserBenefit) — debit
- Активности (Activity Type, Activity, Completion) — credit

Дискриминатор `type: "benefit" | "activity"` разделяет семантику.
`billing_direction: debit | credit` явным образом задаёт направление финансовой операции.
