# T0204 — Диаграмма потока данных в философию.md

## Веха

M02-улучшение-документации

## Контекст

В `философия.md` описаны абстракции (льгота, баллы, активности, категории),
но не показано, как они связаны между собой. Читатель видит 4 абстракции,
но не понимает поток: откуда баллы → куда тратятся → как генерируются.

## Что сделать

Добавить ASCII-диаграмму потока данных в `философия.md` после раздела «Не иерархия. Платформа.»

### Диаграмма

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          ИСТОЧНИКИ БАЛЛОВ                               │
│                                                                         │
│  Billing Rules (credit):                                                │
│    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │
│    │ Ежемесячная  │  │ Бонус за     │  │ Активность   │  │ Реферал  │  │
│    │ по грейдам   │  │ стаж/юбилей  │  │ опрос/ивент  │  │ программа │  │
│    └──────┬───────┘  └──────┬───────┘  └──────┬───────┘  └────┬─────┘  │
│           │                  │                 │               │         │
└───────────┼──────────────────┼─────────────────┼───────────────┼─────────┘
            │                  │                 │               │
            ▼                  ▼                 ▼               ▼
    ┌─────────────────────────────────────────────────────────────┐
    │                    БАЛАНС ПОЛЬЗОВАТЕЛЯ                      │
    │              (currency: points | rub | credits)             │
    └─────────────────────────────────────────────────────────────┘
            │
            │  списание (debit) при подключении льготы
            ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                         СТОИМОСТЬЛЬГОТ                                   │
│                                                                         │
│  Billing Rules (debit):                                                 │
│    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  ┌──────────┐  │
│    │ Фиксированная│  │ % от         │  │ Периодическая│  │ Софинанс │  │
│    │              │  │ benefit_cost │  │ подписка     │  │ 70/30%   │  │
│    └──────────────┘  └──────────────┘  └──────────────┘  └──────────┘  │
└─────────────────────────────────────────────────────────────────────────┘
```

### Текст под диаграммой

```markdown
**Как читать:**
- **Верх** — источники начисления (credit). Каждый — Billing Rule с формулой и триггером.
- **Середина** — баланс пользователя. Валюта настраивается per tenant.
- **Низ** — способы списания (debit). Каждый — Billing Rule, привязанный к льготе.
- Всё — конфигурация. Новый источник/списание = новое правило, не новый код.
```

## Критерии приёмки

- [ ] Диаграмма добавлена в `философия.md` после раздела «Не иерархия. Платформа.»
- [ ] ASCII-диаграмма читается в моноширинном шрифте
- [ ] Текст под диаграммой объясняет 3 уровня
- [ ] Связь с Billing Rule явная
