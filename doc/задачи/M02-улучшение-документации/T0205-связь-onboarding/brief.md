# T0205 — Укрепить связь философия.md ↔ tenant-onboarding.md

## Веха

M02-улучшение-документации

## Контекст

В `философия.md` есть раздел «Как добавить новую компанию (кратко)» со списком из 7 шагов.
В конце — ссылка: `Подробный walkthrough → спецификация/journeys/tenant-onboarding.md`.

Эта ссылка — относительная, без форматирования гиперссылки (просто текст с →).
`tenant-onboarding.md` — подробный документ на 280+ строк с 8 шагами, оценкой времени,
SQL, CSS, проверками. Он заслуживает более заметной ссылки.

## Что сделать

### 1. Улучшить ссылку в `философия.md`

Заменить текущую строку:
```markdown
Подробный walkthrough → `спецификация/journeys/tenant-onboarding.md`.
```

На:
```markdown
> **Полный walkthrough добавления компании (≈2 часа, без изменения кода):**
> [`спецификация/journeys/tenant-onboarding.md`](../спецификация/journeys/tenant-onboarding.md)
>
> 8 шагов: tenant → Keycloak → CSS → биллинг → провайдеры → каталог → активности → импорт → тест.
```

### 2. Добавить обратную ссылку в `tenant-onboarding.md`

В начало `tenant-onboarding.md`, после заголовка, добавить:
```markdown
> **Контекст:** [`контекст/философия.md`](../контекст/философия.md) — «Как добавить новую компанию (кратко)».
```

### 3. Добавить блок в `настраиваемость.md`

В конец `настраиваемость.md`, после раздела «Модели активностей», добавить:
```markdown
---

## Онбординг нового tenant'а

Полный пошаговый процесс добавления компании:
[`спецификация/journeys/tenant-onboarding.md`](../спецификация/journeys/tenant-onboarding.md)

Оценочное время: **2 часа**. Требуемая роль: Admin + HR + Integration Admin.
```

## Критерии приёмки

- [ ] `философия.md` — улучшенная ссылка с preview шагов
- [ ] `tenant-onboarding.md` — обратная ссылка на философия.md
- [ ] `настраиваемость.md` — ссылка на onboarding в конце
- [ ] Все ссылки рабочие (относительные пути)
