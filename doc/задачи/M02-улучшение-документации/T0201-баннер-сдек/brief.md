# T0201 — Баннер «СДЭК — референс» в ключевых документах

## Веха

M02-улучшение-документации

## Контекст

СДЭК упоминается в 8+ документах (проблема.md, акторы.md, journeys/сотрудник.md, journeys/hr.md, модули.md и др.).
Читатель, открывающий эти файлы без философия.md, может ошибочно решить, что платформа — портал СДЭК.

`философия.md` содержит хороший раздел «СДЭК в документации» (таблица «Как читать»),
но он читается только при целенаправленном поиске.

## Что сделать

Добавить компактный блок-предупреждение в начало файлов, где СДЭК-данные появляются первыми:

### 1. `глоссарий.md` — добавить в начало (после заголовка)

```markdown
> ⚠️ **Примеры в документации используют бренд СДЭК.** Это референс для конкретики — НЕ ограничение платформы.
> Платформа — white-label, multi-tenant. СДЭК — первый tenant.
> См. [`контекст/философия.md`](../контекст/философия.md) — раздел «СДЭК в документации».
```

### 2. `контекст/проблема.md` — добавить в начало (после заголовка)

То же предупреждение. Это первый длинный документ, где СДЭК появляется в таблицах.

### 3. `спецификация/journeys/сотрудник.md` — добавить в начало

СДЭК-данные в journey-описаниях.

### 4. `спецификация/journeys/hr.md` — добавить в начало

СДЭК-данные в HR journey.

### 5. `спецификация/journeys/менеджер-каталога.md` — добавить в начало

СДЭК-данные в journey менеджера каталога.

### Формат блока

Единый формат для всех файлов — блокquote с 3 строками:
1. Предупреждение про СДЭК
2. Уточнение white-label
3. Ссылка на философия.md

**Не создавать новый документ** — только добавлять существующим файлам.
