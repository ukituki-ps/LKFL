# M02 — Улучшение видимости white-label архитектуры

## Цель

Сделать очевидным, что платформа — **white-label, multi-tenant, конфигурируемая без кода**.
Читатель документации должен сразу понять: «Это не портал СДЭК — это платформа для любой компании».

## Описание

Анализ документации показал, что архитектура white-label проработана (ADR, биллинг-движок, активности),
но **неочевидна** при первом чтении. СДЭК «шумит» в 8+ документах. Нет единой точки входа для ответа
на вопрос «что настраивается без кода?». Нет data schema для benefit-сущности.
