# T0206 — UI-схема создания Activity Type в активности.md

## Веха

M02-улучшение-документации

## Контекст

В `архитектура/активности.md` есть YAML-schema Activity Type и текстовое описание
«Создание нового типа» (5 шагов). Но не показано, **как выглядит UI-форма**
для менеджера в ЛК-2.

Для сравнения: Billing Rule описан в биллинг-движок.md с примерами моделей.
Activity Type — только схема + текст.

## Что сделать

Добавить wireframe-схему UI-формы создания Activity Type в `архитектура/активности.md`
после раздела «Создание нового типа».

### Wireframe (ASCII)

```
┌─────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Основное                                                                   │
│  ┌─────────────────────────────────────────────────────────────────────┐    │

│  │ UI-компонент:       [ SurveyForm                  v ]               │    │
```

### Текст под wireframe

```markdown
**Валидация при сохранении:**
- `slug` — unique per tenant, lowercase, no spaces
- `completion_criteria` — required, valid JSON для custom типов
- `reward.billing_rule_id` — required, должен существовать и быть active
- `cooldown` — duration string (1h, 7d, 30d) или пустая строка (no cooldown)
- `max_completions` — integer или "unlimited"
```
