# M07 — Аудит архитектуры и рефакторинг модулей

## Описание

Результат архитектурного аудита всех сервисов платформы (см. отчёт по сервисам).
Каждый сервис анализируется на предмет нарушения SRP, избыточных зависимостей, противоречий в документации.

### Что не так

| Проблема | Где | Критичность |
|--|--|--|
| engagement/ — God Object: 4 обязанности в одном пакете | Platform | 🔴 Высокая |
| admin_handler.go — делегирует во ВСЕ пакеты | Platform | 🔴 Высокая |
| providers/ — 11 модулей вместо generic adapter | Integrations | 🔴 Высокая |
| payments/ в integrations/ — без PCI DSS изоляции | Integrations | 🔴 Высокая |
| consent/ — cascade + audit смешаны с grant/revoke | Platform | 🟡 Средняя |
| NATS subjects рассинхронизированы между документами | Документация | 🟡 Средняя |
| Integrations Hub API — концептуален, нет spec | Документация | 🟡 Средняя |
| DmsWizard + MatCapitalWizard — жёстко под СДЭК | Frontend | 🟡 Средняя |

### Что делается

Каждая подзадача M07:
1. Меняет **только документацию** — код не трогается
2. Обновляет все затронутые файлы документации для консистентности
3. Имеет чёткие файлы-мишени с указанием путей

### Правила документирования

Каждая задача ОБЯЗАНА обновить:
- Все файлы `архитектура/`, которые ссылаются на изменённый модуль
- `спецификация/api.md`, если меняются endpoints или contracts
- `контекст/настраиваемость.md`, если меняется матрица
- `архитектура/README.md` — таблица содержимого + ADR список
- `контекст/акторы.md`, если меняются роли взаимодействия
- Создать ADR, если решение принципиально новое

## Волны выполнения

Задачи делятся на 3 волны. Внутри волны — параллельно. Между волнами — строго последовательно.

### Wave A (независимые задачи)

```
T0701 ──────────────┐
T0702 ──────────────┤
T0703 ──────────────┤
T0704 ──────────────┤
T0706 ──────────────┤
T0708 ──────────────┘
```

Все 6 задач свободны от взаимных зависимостей. Могут выполняться параллельно.

**Внимание — `архитектура/пакеты-platform.md`:**
- T0701 пишет «8 пакетов вместо 7» (добавляет `eligibility/`)
- T0702 пишет «9 пакетов вместо 8» (добавляет `compliance/`)
- T0703 пишет новую таблицу handler'ов DI графа
- **Правило:** выполнить последовательно T0701 → T0702 → T0703 (не параллельно между собой), чтобы счётчик пакетов не рассинхронизировался

**Внимание — `архитектура/модули.md`:**
- T0701 меняет число пакетов Platform
- T0703 добавляет comment о разделении admin
- T0704 меняет структуру providers/
- **Правило:** T0701 → T0703 → T0704 (последовательно)

**Остальные файлы (малый overlap):**
- `спецификация/api.md`: T0701 (eligibility endpoints), T0703 (admin group by domain), T0706 (wizard endpoints), T0708 (Integrations Hub) — все добавляют новый раздел, conflict unlikely
- `архитектура/README.md`: все добавляют по 1 строке ADR-таблицы — append safe

### Wave B (после Wave A)


| Задача | Почему после Wave A | Зависимость |
