## Что сделано

Создана полная инфраструктура нагрузочного тестирования для backend LKFL с помощью k6.

### Созданные файлы

| Файл | Описание | Статус |
|------|----------|--------|
| `loadtest/catalog.js` | Каталог: 500 VU, P95 < 200ms, P99 < 500ms | ✅ |
| `loadtest/auth.js` | Auth: 100 VU, P95 < 500ms, P99 < 1000ms | ✅ |
| `loadtest/profile.js` | Profile: 200 VU, P95 < 100ms, P99 < 200ms | ✅ |
| `loadtest/combined.js` | Комбинированная: 600 VU, общий P95 < 300ms | ✅ |

| `loadtest/README.md` | Документация: установка, запуск, CI/CD, интерпретация | ✅ |
| `loadtest/results/` | Директория для результатов | ✅ |

### Детали скриптов

**catalog.js** — 4 сценария с весами:
- Список каталога с пагинацией (60%)
- Детали engagement (20%)
- Фильтрация по типу/статусу (10%)
- Поиск по ключевым словам (10%)
- Ramp: 0→100→500→200→0 VU за 2.5 мин

**auth.js** — 3 сценария:
- GET /user/me — проверка сессии (80%)
- GET /auth/callback — OIDC callback (15%)
- POST /auth/register — регистрация (5%)
- Ramp: 0→20→100→0 VU за 1.5 мин

**profile.js** — 2 сценария:
- GET /user/me — чтение профиля (90%)
- PUT /user/me — обновление контактов (10%)
- Ramp: 0→50→200→0 VU за 1.5 мин

**combined.js** — 6 сценариев (реальное распределение трафика):
- Каталог 40%, Profile 20%, User Engagements 20%, Dashboard 10%, Auth 5%, Recommendations 5%
- Ramp: 0→50→300→600→300→100→0 VU за 5 мин

### run.sh

- Проверка k6 и доступности сервера (healthz)
- Запуск всех 4 тестов с JSON output
- Генерация HTML отчёта с цветовой индикацией pass/fail
- Поддержка запуска отдельных тестов: `./run.sh catalog`
- Кастомный URL через переменную `BASE_URL`
- Exit code отражает результат threshold'ов

### Thresholds

Все скрипты настроены с auto-fail thresholds. k6 вернёт exit code 1 при нарушении SLA:

| Скрипт | P95 | P99 | Ошибки |
|--------|-----|-----|--------|
| catalog | < 200ms | < 500ms | < 1% |
| auth | < 500ms | < 1000ms | < 1% |
| profile | < 100ms | < 200ms | < 1% |
| combined | < 300ms | < 800ms | < 1% |

### Кастомные метрики

Каждый скрипт экспортирует кастомные Trend/Rate/Counter метрики для детального анализа по сценариям.

### CI/CD

README.md содержит примеры интеграции для GitHub Actions и GitLab CI.

## Замечания

1. **Тесты без JWT** — скрипты работают без аутентификации. Для тестов с валидными токенами требуется setup фаза с получением JWT через Keycloak.
2. **Backend должен быть запущен** — run.sh проверяет healthz, но тесты имеют смысл только при работающем lkfl-server с данными.
3. **Threshold'ы ориентировочные** — реальные значения зависят от hardware и конфигурации БД. При первом запуске может потребоваться калибровка.
