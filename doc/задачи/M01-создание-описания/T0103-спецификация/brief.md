### 2. `journeys.md`

User journeys:

**J01 — Регистрация нового сотрудника**
1. HR выгружает XLSX → FTP
2. Админ скачивает → загружает в платформу
3. Платформа валидирует → активирует пользователя
4. Пользователь получает email → переходит по ссылке
5. Пользователь регистрируется (email + ДР)
6. Пользователь входит в систему

**J02 — Распределение льгот**
1. Пользователь входит → видит баланс
2. Открывает каталог → выбирает льготу
3. Открывает детальную модалку
4. Для ДМС: проходит wizard (опция → оплата → подтверждение)
5. Льгота активируется → баланс обновляется

**J03 — Администрирование**
1. Админ загружает Excel-реестр
2. Платформа валидирует → добавляет пользователей
3. Админ выгружает отчёты
4. Админ управляет периодами распределения

**J04 — Интеграция с провайдерами**
1. Пользователь переходит на ready4.ru / PrimeZone
2. Сквозная авторизация (без повторного входа)
3. Использование сервиса провайдера
