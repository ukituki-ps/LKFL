# T0101 — Отчёт о выполнении

## Статус

выполнено

## Что сделано

Созданы 4 файла в `doc/контекст/`:

### 1. `проблема.md` (349 строк)

Бизнес-проблема и цели платформы. 12 проблем, сгруппированных по 3 акторам:
- Сотрудник (5): нет единого входа, фиксированный пакет, родственники — бумага, оплата апгрейда, нет видимости баланса
- HR (3): нет аналитики, документооборот разрозненный, нет канала поддержки
- Компания (4): комплаенс-риски, маткапитал — бумага, white-label vs разовая разработка, нет мобильного

Каждая проблема привязана к элементу прототипа и подтверждена источником (ФТ §, справочник).

### 2. `акторы.md` (~200 строк)

8 акторов в 3 группах:
- Люди (3): Сотрудник, HR-менеджер, Администратор платформы
- Системы (3): Платформа льгот, Кадровая система, Канал передачи данных (брокер)
- Внешние (2): Провайдеры льгот, Пэйшлюз / 1С

Каждый актор описан через: роль, действия, данные, взаимодействие. Матрица взаимодействий (10 ключевых потоков данных).

### 3. `ограничения.md` (~200 строк)

11 нефункциональных ограничений с подробным разбором:
1. Масштаб (100K+ пользователей, горизонтальное масштабирование)
2. Безопасность (OWASP, шифрование, сканирование)
3. Отказоустойчивость (резервирование БД, балансировка, sticky sessions, кластер)
4. Комплаенс (ФСТЭК, 152-ФЗ, единое место хранения)
5. ПДн (lifecycle согласий, data minimization)
6. Браузеры (desktop + mobile, cross-browser)
7. Сроки (лето 2026 — август 2026, MVP-фазировка)
8. Уведомления (push/email/in-app)
9. Журналирование (centralized logging, audit trail)
10. Интеграции (API gateway, message broker, retry logic)
11. Геолокация (J39, 152-ФЗ, Browser Geolocation API)

### 4. `negative-criteria.md` (~180 строк)

10 negative criteria с обоснованием:
1. Не monolith — SPA + модульная архитектура
2. Не храним полисы на платформе — в ядре, по запросу
3. Не ПДн без согласия — consent-first архитектура
4. Не задача без brief → plan → report — правило системы задач
5. Не меняем номенклатуру — RULES.md §1
6. Не привязка к бренду — white-label, multi-tenant
7. Не ручной документооборот — digital lifecycle
8. Не прямая интеграция с кадровой — через брокер
9. Не монолитная БД — master-slave replication
10. Не разовые порталы — универсальная платформа
