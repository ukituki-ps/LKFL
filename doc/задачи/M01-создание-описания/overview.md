# M01 — Создание описания проекта

## Цель

Сформировать полноценную документацию по всем разделам проекта «Платформа гибких льгот» на основе исходных артефактов заказчика.

## Описание

Веха M01 — первая веха проекта. Задача: перевести артефакты заказчика (функциональные требования, прототип, процесс регистрации, коммерческое предложение «Кубики») в структурированную документацию проекта.

## Исходные материалы

Все артефакты в `doc/артефакты/`. Кросс-ссылки в `doc/контекст/README.md`.

| # | Артефакт | Тип |
|---|----------|-----|
| 1 | Функциональные требования v1 | docx |
| 2 | Прототип ЛК физлица | html |
| 3 | Процесс регистрации пользователей | pdf |
| 4 | Единый портал «Кубики» — стоимость | pdf |

## Задачи вехи

| Задача | Раздел | Описание |
|--------|--------|----------|
| T0101-контекст | Контекст | Формирование раздела: проблема, акторы, ограничения, negative criteria |
| T0102-архитектура | Архитектура | Формирование раздела: модули, стек, интеграции, ADR, безопасность |
| T0103-спецификация | Спецификация | Формирование раздела: артефакты, journeys, API, criteria |
| T0104-план | План | Формирование раздела: milestones, tasks, dependencies, exit criteria |

## Exit criteria

- [ ] Все 4 раздела заполнены (контекст, архитектура, спецификация, план)
- [ ] Каждый раздел содержит файлы из таблицы в README
- [ ] Задачи T0101→T0104 имеют заполненные report.md
- [ ] Документация согласована с артефактами заказчика
