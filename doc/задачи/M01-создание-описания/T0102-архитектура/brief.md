# T0102 — Формирование раздела «Архитектура»

## Веха

M01-создание-описания

## Раздел

`doc/архитектура/`

## Контекст

Раздел «Архитектура» описывает техническое проектирование системы. На его основе создаются задачи для реализации. Сейчас в `doc/архитектура/` только README.md с описанием структуры. Нужно создать полноценные файлы раздела.

## Источник данных

- **Существующий код:** `backend/` (Go), `frontend/` (React), `docker/`
- **Функциональные требования v1** — разделы 5-7 (нефункциональные требования, техническая схема)
- **Прототип ЛК** — для понимания frontend-структуры
- **go.mod** — текущие зависимости

## Что сделать

Создать файлы в `doc/архитектура/`:

### 1. `модули.md`

Описать модульную структуру:
- **Backend (Go):** auth, user, document, notification, app, db, middleware
- **Frontend (React):** страницы, компоненты, состояние
- **БД:** PostgreSQL — схема, миграции
- **Кэш:** Redis — сессии, кэш данных
- **Внешние сервисы:** ready4.ru, PrimeZone, Digest, Legal

Для каждого модуля:
- Назначение
- Публичный API
- Зависимости от других модулей

### 2. `стек.md`

Описать технологии:

| Компонент | Технология | Версия | Зачем |
|-----------|-----------|--------|-------|
| Backend | Go | 1.22 | Основная платформа |
| Router | gorilla/mux | 1.8.1 | HTTP-роутинг |
| БД | PostgreSQL | — | Хранение данных |
| Driver | lib/pq | 1.10.9 | Go → PostgreSQL |
| Кэш | Redis | — | Сессии, кэш |
| Redis client | go-redis | 9.7.0 | Go → Redis |
| JWT | golang-jwt | 5.2.1 | Аутентификация |
| CORS | rs/cors | 1.11.1 | Cross-origin |
| Config | spf13/viper | 1.19.0 | Конфигурация |
| Crypto | golang.org/x/crypto | 0.28.0 | Шифрование |
| Frontend | React | — | SPA |

### 3. `интеграции.md`

Описать внешние системы:
- **ready4.ru** — Digest, Legal, основной сайт (сквозная авторизация)
- **PrimeZone** — внешний сервис
- **Подарок в квадрате** — сервис подарков
- **Кадровая система** — выгрузка/загрузка данных
- **Бухгалтерия** — проброс данных
- **FTP Сервер** — ежедневная выгрузка XLSX из HR-системы Заказчика

Для каждой интеграции:
- Протокол (REST, FTP, SSO)
- Формат данных
- Частота взаимодействия
- Обработка ошибок

### 4. `adr/`

Создать папку для ADR (Architecture Decision Records). Первичные ADR:

| ADR | Тема | Статус |
|-----|------|--------|
| 001 | Выбор Go + React для реализации | accepted |
| 002 | PostgreSQL как основная БД | accepted |
| 003 | JWT для аутентификации | accepted |
| 004 | Redis для сессий и кэша | accepted |

### 5. `безопасность.md`

Описать требования безопасности:
- **OWASP Top 10** — защита от уязвимостей
- **Шифрование данных** — при хранении и передаче
- **Валидация паролей** — требования к сложности
- **Сканирование уязвимостей** — регулярное
- **152-ФЗ** — обработка персональных данных
- **ФСТЭК** — сертификация
- **ПДн** — согласие пользователя, удаление по запросу
