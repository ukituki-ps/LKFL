# T1501 — Маппинг прототип → архитектура

## Веха

M15-архитектура-фронтенда

## Контекст

Перед написанием `фронтенд.md` нужно убедиться, что вся документация согласована с прототипом. Прототип — источник истины для UI.

## Что сделать

1. **Сверить страницы** — пройти по всем секциям прототипа (`page-dashboard`, `page-catalog`, `page-points`, `page-documents`, `page-support`) и сверить с таблицей страниц в `архитектура/модули.md` строка 177
2. **Сверить модалки** — DMS wizard, BenefitDetail, MatCapital wizard → сверить с таблицей модалок в `модули.md` строка 193
3. **Описать admin-страницы** — которых нет в прототипе: HR, Catalog, Content (3 страницы). Для каждой: layout, компоненты, данные. **IntegrationsAdmin** — не описывать (часть CatalogAdmin). **RecommendationsAdmin** — не описывать (stub, `модули.md` строка 186: `⏸️ TODO — M15`)
4. **Выявить недостающие компоненты** в tree `src/` — есть ли `Header`, `Footer`, `Sidebar`, `NotificationsPanel`? Если нет — добавить
5. **Зафиксировать расхождения** — если прототип показывает что-то, чего нет в документации (или наоборот)

## Результат

- Список расхождений (если есть)
- Дополненное дерево `src/` (готовое для включения в T1502)
- Описание 3 admin-страниц (HR, Catalog, Content)

## Критерии приёмки

- [ ] Все 5 страниц сотрудника проверены
- [ ] Все 3 модалки проверены
- [ ] 3 admin-страницы описаны (HR, Catalog, Content)
- [ ] Tree `src/` дополнен недостающими компонентами
