# Деплой

Архитектура CI/CD платформы LKFL.

## Архитектура

```
GitHub (push / PR)
  │
  │ .github/workflows/build.yml
  │   ├── lint + test          → ubuntu-latest (public runner)
  │   ├── docker buildx        → lkfl (serverAI, self-hosted runner)
  │   │   └── push → ghcr.io/ukituki-ps/lkfl/{service}:{tag}
  │   │
  │   .github/workflows/deploy.yml (только main)
  │     └── POST → deploy-webhook/deploy → serverAI:9091
  │
  ▼
serverAI — Deploy Worker
  ├── pull из GHCR
  ├── migrate (one-shot)
  ├── up (docker compose)
  ├── seed (one-shot)
  └── healthcheck
```

## Docker Compose

### docker-compose.dev.yml — локальная разработка

- `build:` для lkfl-server и lkfl-integration-proxy
- Volume mount `./frontend/dist/` для nginx
- Все сервисы: postgres, redis, keycloak, server, proxy, nginx
- Без monitoring (prometheus/grafana/loki)

Запуск: `make dev-up`

### docker-compose.staging.yml — staging стенд

- `image:` из GHCR (без `build:`)
- Сервисы: postgres, redis, keycloak, server, proxy, frontend, deploy-worker
- One-shot: lkfl-migrate, lkfl-seed
- Monitoring: Docker profile `monitoring`
- **Нет internal nginx** — один external nginx на сервере, три порта напрямую (ADR-037)
- Архитектура: external nginx (TLS termination) → 18080 (server), 19081 (keycloak), 8081 (frontend)
- `KEYCLOAK_ISSUER: http://keycloak:8080/realms/lkfl-sdek` — внутренний URL (ADR-037)

Запуск: `docker compose -f docker-compose.staging.yml up -d`

### Docker profiles

| Profile | Сервисы |
|---------|---------|
| default | все основные сервисы |
| monitoring | prometheus, grafana, loki, promtail |

Включить monitoring: `docker compose -f docker-compose.staging.yml --profile monitoring up -d`

## Deploy Worker

### API

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/deploy` | Деплой: `{branch, sha, imageTag}` |
| `POST` | `/rollback` | Откат к предыдущему IMAGE_TAG |
| `GET` | `/status` | Текущий статус деплоя |
| `GET` | `/logs` | Логи последнего деплоя |
| `GET` | `/healthz` | Healthcheck |

### Порт

**9091** (не 9090 — конфликт с prometheus!)

### Пайплайн деплоя

1. GHCR login (`docker login ghcr.io`)
2. Pull образов (`docker compose pull --ignore-buildable`)
3. Миграции (`docker compose run --rm lkfl-migrate`)
4. Up сервисов (`docker compose up -d`)
5. Seed (`docker compose run --rm lkfl-seed`)
6. Healthcheck (`curl http://lkfl-server:8080/healthz`)

### Идемпотентность

- Миграции: Atlas idempotent по умолчанию
- Seed: INSERT ON CONFLICT DO NOTHING
- Pull: `--ignore-buildable` (не пересобирать)

## Troubleshooting

### Образы не загружаются (401 Unauthorized)

```bash
# GHCR_TOKEN отсутствует или невалиден
echo "$GHCR_TOKEN" | docker login ghcr.io -u ukituki --password-stdin
```

### Nginx unhealthy (локальная dev)

```bash
# Проверить /nginx-health endpoint (только dev, на staging нет internal nginx)
curl http://localhost:80/nginx-health
# Должен вернуть "ok"
```

> **Staging:** internal nginx удалён (ADR-037). External nginx на сервере обслуживает все три сервиса напрямую по портам 18080, 19081, 8081.

### Deploy Worker недоступен

```bash
# Проверить порт 9091 (не 9090!)
curl http://serverAI:9091/healthz
curl http://serverAI:9091/status
```

### Keycloak callback не работает

- Проверить что `/callback` location proxy на `lkfl_frontend` (не static)
- Проверить upstream `lkfl_frontend` в nginx config

### X-Tenant-ID hardcoded

> ⚠️ **Staging-only limitation:** `X-Tenant-ID sdek` hardcoded в nginx config.
> Для white-label — динамическое разрешение по hostname (M23+).
