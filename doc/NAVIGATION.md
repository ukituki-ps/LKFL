## Критические правила для агентов

1. **Никогда не читай все файлы документации целиком.** Берى только нужный раздел по NAVIGATION.md.
2. **СДЭК в документации — это иллюстрация, не ограничение.** Платформа white-label, multi-tenant.
3. **NATS не используется** (ADR-024). Межмодульная коммуникация — Go interfaces (direct call).
4. **`архитектура/legacy/` — не читать.** Устаревшие файлы.
5. **`архитектура/льготы.md` и `активности.md` — удалены.** Контент в `engagement.md`.
6. **Пакет `recommendations/` — stub (M15, отложено).** Не использовать для новой разработки.
7. **Фронтенд: `AprilProviders` — корневой провайдер.** Не голый `MantineProvider` (фронтенд.md §A).
8. **Фронтенд: иконки — только `AprilIcon`.** Прямой импорт `lucide-react` — запрет (ADR-007).
9. **Фронтенд: DS gap — только 2 реальных gap'а.** Wizard (LKFL обёртка), TransactionList (LKFL компонент) (ADR-029).
10. **Прямые HTTP calls из монолита к провайдерам льгот запрещены — только через Integration Proxy (gRPC).** HR sync (`user/HRSync`) и payments (`internal/payments/`) — исключения. [ADR-035]
11. **Архитектурные риски — всегда записывать в `архитектура/риски.md`.** При аудите, code review, ADR, incident — новый риск → добавить в реестр (формат: §Процесс управления рисками).
