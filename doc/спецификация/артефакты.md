# Артефакты системы

## TL;DR (для агентов)

> Этот файл — **сводная таблица 30 артефактов** (S01–S10, B01, H01–H08, M01–M09, R01, E01). 163 строки.
> - **Какие артефакты и зачем** → строка 10 (§Общая архитектура) — сводная таблица
> - **API для конкретного артефакта** → `api.md` (по названию артефакта в заголовке)
> - **Journeys для артефакта** → `journeys/` (по коду S/B/H/M/R/E)

Сводная таблица 30 артефактов платформы гибких льгот. Каждый артефакт связан с конкретными journeys и API-эндпоинтами.

> Детали полей см. в соответствующих journeys (`journeys/`) и API-контрактах (`api.md`).
>
> **Важно:** все примеры цен, провайдеров и продуктов — из tenant «СДЭК» (первый клиент). Платформа поддерживает N продуктов в M категориях для любого tenant'а. СДЭК — иллюстрация, не ограничение.

## Системные компоненты

| Код | Артефакт | Описание | Journeys | API |
|-----|----------|----------|----------|-----|
| **B01** | Billing Rule | Универсальное правило начисления/списания баллов. direction=credit/debit, amount, category, frequency, trigger | J02, J05, J07, J08, J12, J13, J17, J18, J20a, J21, J26, J34, J35, J43, T01–T07 | `/billing/rules/*` |

**Транзакция** — результат выполнения правила: type (credit/debit), amount, category, source, status (created/frozen/confirmed/cancelled/error), date.

**Счёт пользователя** — агрегация транзакций: total, by_category, expiration_date, days_until_expiration.

## ЛК-1: Кабинет сотрудника

B2C-портал. Фокус: UX, скорость, мобильность.

| Код | Артефакт | Описание | Journeys | API |
|-----|----------|----------|----------|-----|

| S02 | Авторизация | OIDC через Keycloak. Восстановление пароля, auto-login | J01 | Keycloak OIDC |
| S03 | Главная | Витрина: greeting, stat_cards, active_items, event_feed, quick_actions, recommendations, external_links | J01, J02, J05, J11, J12, J13b | `GET /dashboard` |
| S04 | Каталог энгейджментов | Маркетплейс: фильтры по `type=benefit|activity`, категории, поиск. Карточки льгот (type=benefit) + активности (type=activity). seasonal_collections. | J02, J07, J12, J13, J13a | `GET /engagements?type=benefit`, `GET /engagements?type=activity`, `GET /user-engagements` |
| S05 | Баланс | Счёт + категории + прогресс-бары + транзакции + сгорание + предупреждение | J08, J13 | `GET /balance`, `GET /balance/transactions` |
| S06 | Документы | Хранилище PDF: заявления, согласия, полисы. Статусы, скачивание | J05, J06, J07, J10 | `GET /documents`, `GET /documents/:id/download` |
| S07 | Поддержка | FAQ (public, контент из M07) + форма обращения (5 категорий) | J09, J13c | `GET /support/faq`, `POST /support/tickets` |
| S08 | Оплата | Доплата: карта (Visa/MC/МИР) / удержание из ЗП. Заморозка, подтверждение | J08a | `POST /payments/:benefitId`, `POST /payments/:id/confirm` |
| S09 | Engagement Wizard | M07 T0706: единый JSON-driven wizard для любого engagement type. `Wizard.tsx` + `store/wizards.ts`. Новый wizard = JSON config, не React-компонент (ADR-019). type=benefit: рендерит шаги из `engagement_flow.steps` (form → approval → provider_api → document_generation). type=activity: form + condition_check → completion | J02, J04, J05, J06, J07, J12 | `POST /user-engagements`, `GET /user-engagements/:id`, `POST /user-engagements/:id/steps/:stepId/submit`, `POST /user-engagements/:id/completes`, `GET /wizards`, `GET /wizards/:id`, `POST /wizards/:id/validate` |
| S10 | Детали энгейджмента | Модалка: табы динамические — из `engagement_offer.offer_metadata` (description, conditions, policy, clinics). type=benefit: описание льготы + условия. type=activity: описание + бонус + срок. Конфигурируется через offer_metadata, не через код | J02, J03, J07, J12 | `GET /engagements/:id`, `GET /engagement-offers/:id` |

## ЛК-3: Кабинет менеджера каталога

CMS-подобный интерфейс. Фокус: редактирование карточек, дашборды, конфиги.

| Код | Артефакт | Описание | Journeys | API |
|-----|----------|----------|----------|-----|
| M01 | CRUD Engagement Types | name, description, provider_adapter, category, icon, catalog_status, billing_rule_id. type=benefit или type=activity. RBAC guard: hr→activity, catalog_manager→benefit | J21, J22 | `GET/POST/PUT/DELETE /admin/engagement-types` |
| M02 | Метрики каталога | per_item (просмотры, активации, GMV, конверсия), per_category (покрытие, пробелы), aggregate GMV. Применяется к type=benefit engagements | J23 | `GET /admin/engagement-types/analytics` |
| M03 | Продвижение | Баннер, priority_boost, push_notification. Метрики до/после | J25 | `PATCH /admin/engagement-types/:id/catalog-status` (→ seasonal/promo) |
| M04 | Сезонные наборы | Состав из offer-ов (только benefit), период, баннер. Конверсия набора, GMV набора | J27 | `GET/POST/PUT /admin/collections` |
| M05 | Жизненный цикл | 5 состояний: Черновик → Активна → Промо → Скрыта → Завершена. Применяется к engagement-types и engagement-offers | J21–J27, J30 | `PATCH /admin/engagement-types/:id/catalog-status`, `PATCH /admin/engagements/:id/status` |
| M06 | Рекомендации | 2 слоя: контекст (период/сгорание/сезон) + сегмент (грейд/стаж/семья). Debug по userId | J13b, J29 | `GET /recommendations`, `GET/POST /admin/recommendations/rules` |
| M07 | Контент | FAQ CRUD → S07. Баннеры → S03. Описания карточек → S04. **Email-шаблоны** (J44): WYSIWYG-редактор, версионирование, preview. Мгновенная публикация | J28, J44 | `GET/POST/PUT /admin/content/faq`, `/admin/content/banners`, `GET/PUT/DELETE /admin/content/email-templates` |
| M08 | Queue предложений провайдеров | Self-service: сотрудники/HR предлагают новых партнёров. Очередь рассмотрения — менеджер оценивает → создает engagement type (J21) или отказ. Метрика «популярность» предложения | J36 | `GET /admin/provider-proposals`, `POST /admin/provider-proposals/:id/accept`, `POST /admin/provider-proposals/:id/reject` |
| M09 | A/B Testing Engine | Тесты карточек (заголовок, описание, иконка) и баннеров. Расщепление трафика 50/50 или кастом. Метрики: conversion_rate_A/B, significance, winner. Auto-apply winner | J42 | `GET/POST /admin/ab-tests`, `GET /admin/ab-tests/:id/metrics`, `POST /admin/ab-tests/:id/stop` |
