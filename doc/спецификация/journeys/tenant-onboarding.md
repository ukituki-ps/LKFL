## Обзор

Этот документ показывает, как развернуть платформу для новой компании **без изменения кода**.

Оценочное время: **2 часа** для стандартного tenant.

---

## Шаг 1 — Создание tenant'а (10 мин)

### Действия

1. Создать запись в `tenants`:
   ```sql
   INSERT INTO tenants (slug, brand_css) VALUES ('acme', '/theme/brand-acme.css');
   ```

2. Создать Keycloak realm `lkfl-acme`:
   ```bash
   # Через Keycloak Admin CLI или API
   kcadm.sh create realms -s realm=lkfl-acme \
     -s enabled=true \
     -s sslRequired=EXTERNAL
   ```

3. Создать клиентов Keycloak:
   ```
   lkfl-platform, lkfl-billing, lkfl-integrations, lkfl-frontend
   ```

4. Создать роли: `employee`, `hr`, `catalog_manager`, `admin`, `integration_admin`

### Автоматизация

Terraform module выполняет шаги 1-3. Команда:
```bash
terraform apply -var="tenant_slug=acme" -var="tenant_name=ACME Corp"
```

## Шаг 2 — Brand конфигурация (15 мин)

### Действия

1. Загрузить логотип → S3 → записать URL в `tenants.brand_css`
2. Создать CSS-переменные:
   ```css
   :root {
     --brand-color: #1a73e8;
     --brand-color-dark: #1557b0;
     --brand-name: "ACME Benefits";
     --brand-favicon: "/assets/acme/favicon.ico";
   }
   ```

3. Подключить subdomain: `acme.lkfl.io` → Nginx config

### Проверка

Открыть `https://acme.lkfl.io` → увидеть логотип и цвета нового бренда.

## Шаг 3 — Настройка правил биллинга (20 мин)

### Действия

1. Выбрать валюту: `points` / `rub` / `credits` → `tenants.currency_config`
2. Создать Billing Rules:

   **Правило 1 — Ежемесячное начисление:**
   ```
   direction=credit, amount=2000, frequency=monthly, trigger=cron
   condition: status=active
   ```

   **Правило 2 — Бонус за стаж:**
   ```
   direction=credit, amount=1000, frequency=one-time, trigger=event:anniversary
   condition: years_of_service in [3,5,10]
   ```

    **Правило 3 — Бонус за активности (engagement credit):**
    ```
    direction=credit, amount=500, frequency=one-time, trigger=event:engagement_credit
    ```

    **Правило 4 — Списание энгейджмента (engagement debit):**
    ```
    direction=debit, amount=expression:engagement_offer_cost, trigger=event:engagement_debit
    ```

3. Создать первый период распределения:
   ```
   name="Q3 2026", start=2026-07-01, end=2026-09-30
   auto_credit_on_open=true, credit_amount=5000
   ```

### Где

ЛК-2 → «Геймификация» → «Правила начисления» + «Управление периодами»

## Шаг 4 — Подключение провайдеров (30 мин)

### Действия

1. Открыть Integration Proxy admin-панель
2. Для каждого нужного провайдера:

   **Существующий адаптер (большинство):**
   ```
   1. Выбрать адаптер из каталога (AlphaInsurance, WorldClass, YandexFood...)
   2. Ввести endpoints, auth credentials
   3. Протестировать подключение (Health check)
   4. Активировать
   ```

   **Новый провайдер (требует адаптер):**
   ```
   1. Разработать ProviderAdapter (Go-модуль)
   2. Зарегистрировать в Hub
   3. Ввести endpoints, credentials
   4. Протестировать
   ```

3. Настроить кадровую интеграцию:
   ```
   Тип: REST API / XLSX
   Endpoint: https://hr.acme.com/api/employees
   Расписание: ежедневно, 02:00
   ```

4. Настроить пэйшлюз:
   ```
   Provider: Tinkoff / YooKassa / CloudPayments
   Mode: test → production (после валидации)
   ```

### Где

Integration Proxy → Admin UI → «Интеграции»

---

## Шаг 5 — Заполнение каталога (20 мин)

### Действия

1. Создать категории льгот:
   ```
   Здоровье, Спорт, Питание, Обучение, Связь, Мерч
   ```

2. Добавить карточки льгот (через ЛК-3):
   ```
   Для каждой:
   - Название, описание, иконка
   - Провайдер (из подключённых)
   - Стоимость, валюта
   - Категория
   - Условия
   - Billing Rule (debit)
   ```

3. Настроить сезонные наборы (опционально):
   ```
   «Летний набор»: фитнес + питание, период 07.06-31.08
   ```

4. Настроить рекомендации (опционально):
   ```
   Сегмент "Junior" → фитнес + мерч
   Сегмент "Rодители" → ДМС Семья
   Контекст "Период открыт" → баннер на Главной
   ```

5. Заполнить FAQ (минимум 5 вопросов)

### Где

ЛК-3 → «Карточки», «Категории», «Наборы», «Рекомендации», «Контент»

---

## Шаг 6 — Настройка активностей (15 мин)

### Действия

1. Создать типы активностей (если нужны нестандартные):
   ```
   Тип "eNPS опрос": ui_component=SurveyForm, reward=+200
   Тип "Реферал": ui_component=ReferralCard, reward=+300
   ```

2. Создать первые активности:
   ```
   «Опрос удовлетворённости Q3»:
   - Тип: survey, вопросы: 5
   - Награда: +500 баллов
   - Период: 01.07-31.07
   - Аудитория: все сотрудники
   - Уведомления: email + push
   ```

### Где

ЛК-2 → «Геймификация» → «Активности»

---

## Шаг 7 — Импорт сотрудников (5 мин)

### Вариант A — Автоматический

```
1. Настроить endpoint кадровой системы (шаг 4)
2. Запустить первую синхронизацию: manual trigger
3. Проверить валидацию: ошибки → коррекция → повтор
4. Результат: N сотрудников импортированы → email-уведомления отправлены
```

### Вариант B — Ручной

```
1. Скачать XLSX с реестром (HR-система заказчика)
2. Загрузить в ЛК-2 → «Пользователи» → «Импорт»
3. Проверить ошибки (отчёт XLSX)
4. Результат: сотрудники импортированы → email-уведомления
```

## Шаг 8 — Тестовый прогон (20 мин)

### Сценарий

1. **Регистрация** → перейти по email-ссылке → заполнить профиль → войти
2. **Главная** → увидеть баланс, активные льготы, рекомендации, баннеры
3. **Каталог** → открыть карточку льготы → подключить → проверить списание
4. **Баланс** → увидеть транзакцию списания, обновлённый остаток
5. **Активность** → найти доступную активность → выполнить → получить баллы
6. **ДМС Wizard** → апгрейд → выбор оплаты → подтверждение → статус «Ожидает»
7. **Документы** → проверить наличие сгенерированных документов
8. **Поддержка** → проверить FAQ, создать тестовое обращение

### Проверка

| Проверка | Ожидаемый результат |
|----------|-------------------|
| Бренд | Логотип, цвета, название — нового tenant'а |
| Баланс | Начисление +5000 при открытии периода |
| Льготы | Каталог отображает подключённые провайдеры |
| Списание | Подключение льготы → транзакция debit |
| Активность | Выполнение → транзакция credit |
| Нотификации | Email при регистрации, push при начислении |
| Документы | PDF-заявление сгенерировано |
| Multi-tenancy | Данные изолированы (tenant_id в запросах) |
