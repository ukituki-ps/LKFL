## Изоляция тестирования по пакетам (M06)

> **Контекст:** lkfl-server (16 internal-пакетов монолита + integration-proxy). Каждый бизнес-пакет должен иметь unit-тесты без запуска Platform целиком.

| # | Критерий | Пакет | Как проверить |
|---|--|--|-|
| 58 | Test isolation: eligibility engine | `internal/eligibility/engine.go` | Unit-test `EligibilityEngine.EvaluateCEL()` с mock `CELContext` — 20+ test cases (ADR-021: CEL expression evaluation). Check() с mock `UserProfile` — AND/OR legacy + CEL |
| 59 | Test isolation: Channel interface | `internal/notification/channels.go` | Unit-test `Channel` implementation через mock channel — Send() возвращает nil error, payload validation |
| 60 | ⏸️ Test isolation: recommendations evaluator | `internal/recommendations/evaluation.go` | ⏸️ Отложено до M15 Phase 2. Stub `internal/recommendations/` возвращает `[]Recommendation{}`. Unit-test `EvaluationEngine.Evaluate()` с mock `UserProfile` — context rules + segment rules scoring |
| 61 | Test isolation: flow engine | `internal/engagement/flow/flow_engine.go` | Unit-test `FlowEngine.Activate()` → верный status transition → direct Go call (M12) |
| 62 | Test isolation: collections engine | `internal/engagement/collections/collection_engine.go` | Unit-test `CollectionsEngine.Engage()` → create N user_engagements → validate bundle_price |
