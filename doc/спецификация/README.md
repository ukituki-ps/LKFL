# Спецификация

> **🗺️ Навигация:** [`doc/NAVIGATION.md`](../NAVIGATION.md) — карта «вопрос → файл:строка»

Этот раздел описывает **артефакты**, **user journeys**, **API-контракты** и **критерии приёмки** проекта.

## Назначение

Спецификация определяет, что система делает с точки зрения пользователя и внешних систем. Она отвечает на вопросы:
- **Какие артефакты создаём?** — личный кабинет, каталог льгот, ДМС wizard, панель администратора, массовые уведомления, аудит-лог, A/B тестирование
- **Как пользователь взаимодействует с системой?** — 57 journeys от регистрации до распределения льгот, массовых рассылок, командных челленджей
- **Какие API предоставляем?** — 118 REST API endpoints для frontend и внешних систем
- **Когда считаем задачу выполненной?** — 66 exit criterion для каждой вехи

## Основные user journeys

1. **Регистрация сотрудника** — синхронизация из HR через REST API → валидация → email-уведомление → регистрация по email+ДР
2. **Распределение льгот** — вход → каталог → выбор опций → ДМС wizard → подтверждение → баланс
3. **Администрирование** — загрузка реестра Excel → валидация → активация пользователей → отчёты
4. **Интеграция с провайдерами** — сквозная авторизация → digest.ready4.ru, legal.ready4.ru, PrimeZone
