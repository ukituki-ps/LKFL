## A. Обзор

**Назначение.** Пользовательский интерфейс: личный кабинет сотрудника + админ-панель. White-label multi-tenant платформа. 100 000+ сотрудников.

**Технологии:** Vite + React 18 + `@april/ui` + `@april/tokens` + Mantine + React Query.

**ADR:**
- [ADR-007](./adr/007-april-ui.md) — April UI + Mantine foundation
- [ADR-008](./adr/008-white-label.md) — White-label через CSS переменные
- [ADR-012](./adr/012-zustand.md) — Zustand для state management
- [ADR-029](./adr/029-ds-components-gap-tz.md) — DS gap analysis (11 → 2 gap'а)
- [ADR-031](./adr/031-api-data-fetching.md) — React Query (data fetching)
- [ADR-032](./adr/032-api-types-codegen.md) — OpenAPI codegen для types
- [ADR-033](./adr/033-frontend-testing.md) — Vitest + Playwright
- [ADR-034](./adr/034-i18n-yagni.md) — i18n YAGNI

### DS-интеграция

**`AprilProviders`** — корневой провайдер (не голый `MantineProvider`):
```tsx
// main.tsx
import { AprilProviders } from '@april/ui'
import { createAprilTheme } from './theme/theme'

ReactDOM.createRoot(document.getElementById('root')!).render(
  <AprilProviders theme={createAprilTheme()}>
    <App />
  </AprilProviders>
)
```

**Что включает `AprilProviders`:**
- `@april/tokens` — CSS variables, цвета, typography
- `MantineProvider` — с April theme override
- `DensityProvider` — comfortable/compact (comfortable по умолчанию)
- `ColorSchemeScript` — light/dark (light по умолчанию)

### Density

- **Comfortable** — по умолчанию. Все компоненты в comfortable mode.
- **Compact** — YAGNI. Не включать без реального use case.
- `useDensity()` — хук для адаптивных размеров компонентов.

### Тёмная тема

- DS поддерживает (light + dark через `useMantineColorScheme()`).
- **YAGNI — не включать по умолчанию.** Первый tenant (СДЭК) — только light.
- Запас: `useMantineColorScheme()` + `MantineColorSchemeProvider` — готово для future.

### Иконки

- **Только `AprilIcon` + `AprilIcon*`** из `@april/ui`.
- Прямой импорт `lucide-react` — **запрет** (зафиксировать в ADR-007 при обновлении).

## D. API Layer

**Стек:** native `fetch` (через Nginx proxy) + React Query (ADR-031) + openapi-typescript (ADR-032).

### Fetch wrapper

```ts
// api/platform.ts
const BASE_URL = '/api/v1'

export async function apiFetch(url: string, options?: RequestInit): Promise<Response> {
  const token = await getToken() // Keycloak token

  const res = await fetch(`${BASE_URL}${url}`, {
    ...options,
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`,
      ...options?.headers,
    },
    signal: AbortSignal.timeout(10_000), // 10s timeout
  })

  if (!res.ok) {
    const error = await res.json().catch(() => ({ message: res.statusText }))
    throw new ApiError(res.status, error)
  }

  return res
}
```

### Error handling

| Код | Поведение |
|-----|-----------|
| 401 | Redirect на Keycloak login |
| 403 | Показать «Доступ запрещён» |
| 422 | Показать валидационные ошибки формы |
| 500+ | Toast «Произошла ошибка. Попробуйте позже» |
| Network | Retry 1× (1s delay) → offline toast |

### Retry policy

- **Retry:** 1 attempt, 1s delay (только network errors)
- **No retry:** 4xx errors (client errors)
- **React Query retry:** `retry: 1`, `retryDelay: 1000`

### Caching

- React Query: `staleTime`, `gcTime` (по кейсам — ADR-031)
- Каталог: `staleTime: 5min` (backend Redis cache sync)
- Баланс: `staleTime: 0` (refetch on window focus)
- Нотификации: `refetchInterval: 30s` (polling)

### Optimistic updates

- Survey submit: optimistic → rollback на error (ADR-031)
- Balance: mutation → `invalidateQueries(['balance'])` → refetch

## E. State Management (Zustand)

**Библиотека:** Zustand 5.0 (ADR-012). Selector-based, typed, devtools middleware.

### Store'ы

| Store | Файл | Данные |
|-------|------|--------|
| User | `store/user.ts` | Профиль, роль, tenant, brand config |
| Balance | `store/balance.ts` | Баланс, категории, транзакции (через React Query + Zustand sync) |
| Catalog | `store/catalog.ts` | Каталог, фильтры, поиск, active category |
| Notifications | `store/notifications.ts` | Список нотификаций, unread count |
| Wizards | `store/wizards.ts` | Active wizard, steps data, navigation |
| Content | `store/content.ts` | FAQ, баннеры, описания (admin) |

### Pattern

```ts
import { create } from 'zustand'

interface BalanceStore {
  balance: number
  categories: Record<string, number>
  isLoading: boolean
  fetchBalance: () => Promise<void>
}

export const useBalanceStore = create<BalanceStore>((set) => ({
  balance: 0,
  categories: {},
  isLoading: false,
  fetchBalance: async () => {
    set({ isLoading: true })
    const data = await apiFetch('/billing/accounts/me/balance').then(r => r.json())
    set({ balance: data.balance, categories: data.categories, isLoading: false })
  },
}))

// Selector pattern — избегать re-render:
const balance = useBalanceStore(state => state.balance)
```

> **Примечание:** React Query (ADR-031) управляет server state. Zustand — UI state (open/closed, filters, wizard steps).

## H. Performance

### Code splitting

- Все страницы lazy-loaded (`React.lazy()` + `Suspense`)
- Admin-страницы — отдельный chunk (`admin/`)
- Модалки — lazy-loaded через `React.lazy()`

### Bundle analysis

```bash
# Vite plugin для анализа
npm run build -- --mode analyze
```

**Target:** <200KB gzipped (initial). Каталог — самый большой chunk (карточки + фильтры).

### Lighthouse targets

| Метрика | Целевое значение |
|---------|-----------------|
| LCP | <2.5s |
| CLS | <0.1 |
| FID | <100ms |
| TTI | <3.5s |

### Error tracking

**Sentry** (`@sentry/react`) — error grouping, source maps, breadcrumbs.

**Инициализация:** `main.tsx` (после `AprilProviders`). DSN из env.
```tsx
import * as Sentry from '@sentry/react'

Sentry.init({
  dsn: import.meta.env.VITE_SENTRY_DSN,
  environment: import.meta.env.VITE_ENVIRONMENT,
  integrations: [
    new Sentry.BrowserTracing({ tracePropagationTargets: ['/api/'] }),
    new Sentry.Replay(),
  ],
  tracesSampleRate: 0.1,
  replaysSessionSampleRate: 0.1,
})
```

### PWA

**`vite-plugin-pwa`** — service worker для каталога льгот (offline reading).

- Manifest + cache strategy для `GET /catalog`
- Offline: каталог льгот доступен без сети (статичные данные)
- Background sync: при появлении сети — refetch
