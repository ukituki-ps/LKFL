# ADR-002: PostgreSQL как основная база данных

**Статус:** Accepted
**Дата:** 2026-05-22
**Контекст:** М01-создание-описания

## Контекст

Необходимо выбрать БД для хранения данных пользователей, льгот, транзакций и интеграций. 100 000+ пользователей, ФСТЭК-сертификация, 152-ФЗ.

## Решение

**PostgreSQL 17** — ACID-гарантии (критично для биллинга), JSONB для гибких конфигов льгот,成熟的 экосистема миграций (Atlas), WAL archiving для DR.

## Альтернативы

| Вариант | Плюсы | Минусы |
|---------|-------|--------|
| MySQL | Проще | Нет JSONB, слабее ACID |
| MongoDB | Гибкая schema | Нет ACID на multi-doc, сложнее ФСТЭК |
| SQLite | Простой deploy | Не кластеризуется, не подходит для 100K+ |

## Следствия

- Три separate БД: `lkfl_platform`, `lkfl_billing`, `lkfl_integrations`
- Atlas для миграций (SQL-first)
- WAL archiving + pg_basebackup для DR
- `sslmode=verify-full` в production
