# Глоссарий

> ⚠️ **Примеры в документации используют бренд СДЭК.** Это референс для конкретики — НЕ ограничение платформы.
> Платформа — white-label, multi-tenant. СДЭК — первый tenant.
> См. [`контекст/философия.md`](./контекст/философия.md) — раздел «СДЭК в документации».

Термины, аббревиатуры и коды, используемые в документации проекта.

## Термины проекта

| Термин | Описание |
|--------|----------|
| Платформа | Ядро системы: UI, биллинг, документооборот, комплаенс, нотификации |
| Кафетерий льгот | Возможность сотрудника распределять бюджет льгот между N продуктами в M категориях (конфигурируемо) |
| Белый ярлык (white-label) | Универсальная платформа: один код, любой бренд. CSS-переменные, конфиг бренда |
| Multi-tenancy | Одна установка платформы для нескольких компаний. Изоляция данных по tenant_id |
| Сквозная авторизация | Переход на внешний сервис без повторного входа (Keycloak Identity Broker → OIDC redirect) |
| Биллинговый движок | Модуль управления балансом пользователя: начисление, списание, периоды |
| Бонусный счёт | Виртуальный баланс баллов сотрудника |
| ДМС | Добровольное медицинское страхование |
| Апгрейд ДМС | Переход с базовой программы на расширенную или премиум |
| Маткапитал | Корпоративная выплата 100 000₽ при рождении ребёнка (стаж ≥ 6 мес, заявка ≤ 90 дней) |
| Геймификация | Система стимулирования: баллы за поведение (опросы, активности, стаж) |
| Активность (Engagement type=activity) | Действие сотрудника для получения бонусных баллов (credit): опрос (N вопросов), корпоративное событие, реферал. Часть единой абстракции Engagement. |
| Сезонный набор | Тематический набор из нескольких льгот с объединённым баннером и периодом действия |
| Биллинг | Финансовый сервис: баланс, транзакции (credit/debit), периоды |
| Промо | Статус карточки льготы: активное продвижение с баннером, приоритетом и push-уведомлением |
| Период распределения | Временное окно (обычно полугодичное), в которое сотрудник выбирает льготы |

| Каскадная деактивация | Цепочка при увольнении: ① блокировка ЛК → ② открепление льгот (сотр. + родс.) → ③ остановка billing rules → ④ запрос баланса партнёрам → ⑤ реестр открепления |
| Consent lifecycle | Цикл согласия ПДн: создание → действие → продление → отзыв → удаление |
| Cursor-based pagination | Пагинация по курсору (base64): `?cursor=xyz&limit=20`. Ответ включает `nextCursor`, `hasMore`, `total`. Альтернатива offset/limit для больших датасетов. |
| Tenant onboarding | Процесс добавления нового tenant'а: создание БД-записи, Keycloak realm, brand CSS, seed-данные. Terraform + admin UI. |
| Data retention | Политика хранения данных: срок хранения каждого типа данных + действие по истечении (архивация, удаление, псевдонимизация). |
| Rate limiting per tenant | Ограничение запросов с учётом tenant_id: `rate_limit:{tenant_id}:{user_id}:{endpoint}`. Один tenant не затопит другой. |
| Апрув | Проверка заявки HR-менеджером: маткапитал, психолог, апгрейд ДМС с удержанием из ЗП |

## Коды задач

| Формат | Описание |
|--------|----------|
| `M{MM}-{slug}` | Веха: номер и краткое название (M01-создание-описания) |
| `T{MM}{NN}-{name}` | Задача: MM — номер вехи, NN — порядковый номер |
| Пример | `T0103-спецификация` = задача 03 из вехи M01 |

## Интеграционный модуль

| Термин | Описание |
|--------|----------|
| Integration Proxy (`lkfl-integration-proxy`) | Отдельный бинарник: gateway к внешним провайдерам льгот. Adapter pattern, circuit breaker per provider, retry/timeout, webhook receiver, credential store. Общается с монолитом через gRPC на localhost. [ADR-035] |
| Integration Client (`internal/integrationclient/`) | gRPC клиент в монолите для вызовов Integration Proxy. Заменяет `internal/integrations/` (M16). |
| Provider Onboarding | Процесс подключения нового провайдера: регистрация → YAML config → адаптер в `integration-proxy/adapters/` → тестирование → публикация. REST-провайдер без кода (M07 T0704). |
| Абстрактный провайдер | Провайдер как конфигурационный объект: name, protocol, endpoints, auth_method. Количество не фиксировано — «сколько есть, столько и будет». |
| Integration Admin | Роль: управляет каталогом провайдеров через Integration Proxy admin endpoints + admin UI ЛК-3. Проверяет статусы, настраивает endpoint'ы, следит за SLA провайдеров. |
