# Зависимости между вехами и задачами

## TL;DR (для агентов)

> Dependency graph между 67 задачами по 17 вехам.
> - **Визуальный граф:** строка 10 — зависимости M00→M16
> - **Критический путь:** M01 → M05 → M07 → M12 → M16
> - **Параллельные ветки:** M08 (CEL/LLM) и M09 (Gamification) не зависят друг от друга
> - **M15:** зависит от M13 (frontend-архитектура использует survey-архитектуру из ADR-025)
> - **M16:** зависит от M12 (Integration Proxy — вынос integrations/ из монолита)
> - **M17:** зависит от M16 (Bootstrap — код-задачи начинаются после завершения архитектуры)

> **Связь:** `план/вехи.md` → перечень вех. `план/критерии-вех.md` → exit criteria. `план/задачи.md` → реестр задач.

## Критические пути

### Путь 1: Фундамент → Унификация

```
M01 → M02 → M03 → M04 → M05 → M07
```

Длина: 6 вех. Нельзя ускорить — каждая веха зависит от предыдущей.

### Путь 2: Рефакторинг → Монолит

```
M07 → [M08, M09, M10, M11] → M12
```

Длина: 3 вехи (M08-11 параллельны). M07 — bottleneck.

### Путь 3: Монолит → Survey

```
M12 → M13
```

Длина: 2 вехи. M14 (Survey Implementation) ⛔ отменена.

### Путь 4: Survey → Frontend architecture

```
M13 → M15
```

Длина: 2 вехи. M15 — frontend-документация после M13 (архитектура survey из ADR-025).

### Путь 5: Монолит → Integration Proxy

```
M12 → M16
```

Длина: 2 вехи. M16 — вынос integrations/ из монолита в отдельный бинарник proxy (ADR-035).

### Общий критический путь

```
M01 → M02 → M03 → M04 → M05 → M07 → [M08||M09||M10||M11] → M12 → [M13||M16] → M15
```

Итого: 6 + 1 + 1 + 1 + 1 = **10 последовательных шагов** (M08-M11 параллельно = 1 шаг; M13+M16 параллельно после M12).
