## M17 — Authorization (первая задача кода)

**Цель:** создать полностью рабочую систему авторизации — от Keycloak до React SPA. Фундамент для всех последующих задач.

### Граф зависимостей задач M17

```
T1701 (инфраструктура)
    ├── T1702 (backend auth) ───┐
    ├── T1703 (frontend auth) ──┼──→ T1704 (CI/CD: Фаза A+B)
    └── T1705 (Observability)   — можно отложить до M18
```

**Можно отложить без ущерба:** T1705 (Observability).
**Критический путь:** T1701(3d) → T1702(5d)/T1703(4d) → T1704(5d) = 13 дней.

### Задачи

| Задача | Тип | Описание | Зависит от | Статус |
|--------|-----|----------|------------|--------|
| T1701 | код | Инфраструктура и bootstrap: go.mod, docker-compose, Makefile, Keycloak template, Nginx | — | ⏳ |
| T1702 | код | Backend auth core (+ unit тесты): shared/pkg/auth, internal, migrations, cmd | T1701 | ⏳ |
| T1703 | код | Frontend auth (+ Playwright E2E): Vite, keycloak-js, Zustand, state machine, layout, E2E | T1701 | ⏳ |
| T1704 | код | CI/CD pipeline: Фаза A (3d: Dockerfile+Actions), Фаза B (2d: testcontainers+OpenAPI) | T1702, T1703 | ⏳ |
| T1705 | код | Observability: Prometheus + Grafana + Loki, дашборды (можно отложить до M18) | T1701 | ⏳ |

**ADR:** ADR-036 (Authorization System), ADR-003 (Keycloak), ADR-009 (Multi-tenancy)

**Exit criteria (сборка из T1701–T1704):**
- `lkfl-server` компилируется и запускается
- Keycloak поднимается в docker-compose с демо realm
- Frontend запускается, keycloak-js инициализируется
- Login flow работает: guest → login → transition → authorized
- `/api/v1/me` возвращает профиль с ролями
- Tenant resolution работает по subdomain
- JWT валидация работает с realm-specific JWKS
- RBAC guard блокирует endpoints без нужных ролей
- `go test ./...` все зелёные
